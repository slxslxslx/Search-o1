`.gitignore` 只对**未跟踪的文件**生效。已经被 `git add` + `commit` 的文件，即使后来加到 `.gitignore`，Git 仍会继续跟踪。

---

## 解决步骤

### 1. 从 Git 中移除（保留本地文件）

```bash
git rm -r --cached cache/
git rm -r --cached log/
```

- `--cached`：只从 Git 索引中移除，**本地文件保留**
- `-r`：递归处理目录

### 2. 提交这次移除

```bash
git commit -m "Remove cache/ and log/ from tracking, add to .gitignore"
```

### 3. 确保 `.gitignore` 已包含

```bash
echo "cache/" >> .gitignore
echo "log/" >> .gitignore
git add .gitignore
git commit -m "Update .gitignore to exclude cache/ and log/"
```

### 4. 推送到远程

```bash
git push
```

---

## 验证

```bash
git ls-files | grep -E "^cache/|^log/"
```

如果没有任何输出，说明 Git 不再跟踪这两个目录。

---

## 完整一键命令

```bash
git rm -r --cached cache/ log/
echo "cache/" >> .gitignore
echo "log/" >> .gitignore
git add .gitignore
git commit -m "Stop tracking cache/ and log/, update .gitignore"
git push
```

---

## 注意

| 情况 | 说明 |
|------|------|
| 远程仓库已有这些文件 | 执行后远程的 `cache/` 和 `log/` 会被删除，但**不影响其他协作者的本地文件** |
| 其他协作者 | 他们下次 `git pull` 时，本地 `cache/` 和 `log/` 保留，但不再被 Git 跟踪 |
| 误删恢复 | 如果操作错了，用 `git checkout HEAD -- cache/` 可以从最近一次提交恢复 |

---

执行完这些步骤后，后续 `cache/` 和 `log/` 下的新文件就不会再被 Git 跟踪了。
